# Ejercicio 03 - HTML

## Introducción
El Lenguaje de Marcado de Hipertexto, conocido universalmente como **HTML** (HyperText Markup Language), es la piedra angular de la World Wide Web. Es el estándar encargado de definir la estructura y el significado del contenido digital. Preparar un documento HTML es, en esencia, organizar la jerarquía de la información para que los navegadores puedan interpretarla y presentarla de manera coherente al usuario final.

## Síntesis
HTML no es un lenguaje de programación en el sentido estricto, sino un **lenguaje de marcado** que utiliza una sintaxis basada en etiquetas (tags) para estructurar datos. Un documento estándar comienza con la declaración `<!DOCTYPE html>`, seguida por la etiqueta raíz `<html>` que envuelve todo el contenido. Esta estructura se divide fundamentalmente en dos partes: el `<head>`, donde se aloja la metadata, el título de la pestaña y los enlaces a recursos externos; y el `<body>`, que contiene todo lo que es visible para el usuario, como textos, imágenes y contenedores.

La evolución hacia el **HTML5** introdujo la importancia de la semántica. Esto significa utilizar etiquetas que describen su propósito, como `<header>`, `<nav>`, `<main>`, `<article>` y `<footer>`. El uso de estas etiquetas en lugar de simples `<div>` genéricos es vital para el SEO (optimización en motores de búsqueda) y, sobre todo, para la accesibilidad web. Al usar una estructura semántica, facilitamos que herramientas como los lectores de pantalla para personas con discapacidad visual puedan navegar e interpretar el sitio de forma lógica y fluida.

Además, HTML permite la integración de elementos multimedia y enlaces mediante atributos específicos. Etiquetas como `<a>` para hipervínculos, `<img>` para imágenes y `<ul>`/`<li>` para listas, permiten que la información deje de ser un bloque de texto plano para convertirse en un sistema hiperconectado. En resumen, HTML proporciona el esqueleto y la organización sobre la cual se apoyan otras tecnologías como CSS para el estilo y JavaScript para la funcionalidad, formando así el ecosistema básico del desarrollo web.

## Reflexión
Lo que considero más útil de HTML es su capacidad para **organizar el caos informativo**. Me parece fascinante que, con unas cuantas reglas de marcado, podamos dar sentido y jerarquía a cualquier tipo de contenido. Aprender sobre la semántica me ha hecho comprender que el desarrollo web no solo se trata de cómo se ven las cosas en pantalla, sino de cómo la información es procesada y entendida por diferentes dispositivos y usuarios, fomentando una red más inclusiva y ordenada.

## Conclusión
En conclusión, HTML es la base inamovible de cualquier proyecto en la web. A través de este ejercicio, se ha sintetizado cómo su estructura de etiquetas y su enfoque semántico permiten construir documentos robustos y accesibles. Sin la estructura que HTML proporciona, la web carecería de orden y significado, reafirmando que, a pesar del surgimiento de nuevas tecnologías, el dominio de este lenguaje de marcado sigue siendo la habilidad fundamental para cualquier desarrollador.
